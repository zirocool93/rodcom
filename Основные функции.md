1. Главная концепция проекта

Система должна решать 5 основных задач:

Учет участников
Школа, классы, ученики, родители, комитет, роли.
Учет сборов и платежей
Кто участвует в сборе, сколько нужно внести, кто оплатил, кто частично оплатил, кому нужен возврат.
Учет расходов и чеков
На что потрачены деньги, кто оплатил, какой чек прикреплен, из какого сбора списано.
Прозрачность и согласования
Голосования, утверждение расходов, история изменений, отчеты.
Удобный доступ
Web/PWA для телефона + Telegram-бот с поддержкой SOCKS5-прокси.
2. Что берем из готовых решений
Из ERPNext / Frappe

Берем идею документной структуры.

Почти всё в системе — это документ:

сбор;
платеж;
расход;
чек;
голосование;
отчет;
заявление на возврат;
мероприятие;

У каждого документа есть:

номер;
дата;
автор;
статус;
история изменений;
комментарии;
вложения;
права доступа.

Пример статусов:

Черновик → На согласовании → Утверждено → Выполнено → Закрыто

Это удобно, потому что потом любую операцию можно отследить.

Модуль 3. Роли и права доступа

Роли лучше сделать гибкими.

Основные роли:
Роль	Возможности
Суперадмин	Управляет всей системой
Админ школы	Управляет школой и классами
Председатель комитета	Создает сборы, голосования, отчеты
Казначей	Ведет платежи, расходы, чеки
Ревизор	Видит всё, но не редактирует
Родитель	Видит свои платежи, сборы, отчеты, голосования
Учитель	Видит мероприятия и общие отчеты, без финансовых деталей
Наблюдатель	Только просмотр разрешенных отчетов

Нужно сделать настройку прав по модулям:

Сборы: смотреть / создавать / редактировать / утверждать / удалять
Платежи: смотреть / добавлять / подтверждать / удалять
Расходы: смотреть / создавать / согласовывать / проводить
Отчеты: смотреть / формировать / публиковать

11. Telegram-бот
Модуль 16. Telegram-интеграция

Telegram-бот должен быть отдельным сервисом, который подключается к основному backend через API.

Важное требование

В настройках Telegram-бота должна быть возможность указать SOCKS5-прокси.

Пример настроек:

BOT_TOKEN=123456:ABCDEF
TELEGRAM_PROXY_ENABLED=true
TELEGRAM_PROXY_TYPE=socks5
TELEGRAM_PROXY_HOST=192.168.31.203
TELEGRAM_PROXY_PORT=2080
TELEGRAM_PROXY_USERNAME=
TELEGRAM_PROXY_PASSWORD=

24. Главные отличия нашего решения

Чтобы проект был не просто копией учета платежей, я бы заложил такие уникальные функции:

Мягкая финансовая лексика
Не “долг”, а “ожидается оплата” или “не поступило подтверждение”.
Приватность платежей
Родители не обязаны видеть платежи других семей.
Чеки и прозрачность расходов
Каждый расход связан с чеком или пометкой “без подтверждения”.
Голосования родителей
Важные решения проходят через согласование.
Telegram-бот с SOCKS5
Это сразу делает продукт удобным в реальных условиях.
PWA вместо отдельного мобильного приложения
Быстрее разработка, проще установка, работает на Android/iOS.
Отчеты одной кнопкой
PDF/Excel для отправки в родительский чат.
Журнал действий
Кто, когда и что изменил.
Ревизорский доступ
Отдельная роль для проверки без права редактирования.
Архив учебного года
В конце года можно закрыть баланс и сохранить архив.
